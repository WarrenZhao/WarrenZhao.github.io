本篇博客的初衷

本人只在大学期间学习过c++，目前在做机器人相关项目，需要使用ros系统，虽然本人喜欢使用Python，但是现有软件是使用c++开发的，所以写一篇博客总结一下c++常用的语法点以及在使用过程中需要注意的问题，以便能够快速上手c++开发。

变量的使用及其作用域

  1. 寄存器变量：寄存器变量用于建议编译器将变量防止在CPU中的寄存器中，以加快访问速度。在定义变量时，使用register关键字表示该变量是一个寄存器变量。寄存器变量不一定保证变量放在寄存器中，也不能保证访问速度，它只是对编译器的一个建议。寄存器变量不能成为全局变量或者静态变量。
  2. 静态变量：在函数体外定义的变量，声明周期随着程序的结束而结束，只执行一次初始化，使用static关键字来定义。
  3. 外部变量：使用extern关键字定义，用于高速编译器在其他文件的全局区域存在一个全局变量，目的是在当前的文件中能够访问这个全局变量。
  4. 自动变量：使用auto关键字进行声明。局部变量默认状态下就是自动变量。自动变量只能在函数内使用，定义全局自动变量是非法的。
  5. wchar_t型变量：字符类型，此类型的长度为16字节，char的长度是8个字节，char字符类型无法完全包括很多的外文字符集。

引入头部文件的方式和区别

第一种：“”

该种方法引用的头文件一般是自定义的头文件，引用的是程序目录相对路径中的头文件。

比如：项目的目录是在D:\Program\tmp\，则#include “my.h”引用的就是D:\Progam\tem\my.h这个文件

第二种：<>

该种方法引用的是编译器的类库路径里的头文件，无论项目的目录是哪个，只要编译器路径不变更即可。

如何使用终端编译、执行c++程序

操作系统：macos
由于mac是自带g++编译器的，所以使用终端编译只需要先编辑好cpp文件（例如：hello.cpp），然后打开终端，cd到需要编译的cpp文件目录下面，然后执行g++ -o hello hello.cpp如果程序没有错误，则会直接在当前目录下面生成一个叫做hello的可执行文件，使用命令./hello即可执行编译好的c++程序。或者直接使用命令g++ hello.cpp这种方式下，编译好的文件名称是a.out使用./a.out即可执行程序。

栈和堆

对于c++来说，程序中的数据主要有两种存储形式，一种是栈存储方式，一种是堆存储方式。栈存储通常用于存储占用空间小、生命周期短的数据，例如局部变量和函数参数等；堆存储通常用于存储占用空间大、生命周期长的数据，例如静态变量和全局变量等。

如何在堆中申请空间？

使用new关键字可以在堆中开辟一块空间，但是需要使用delete关键字释放申请的空间否则会出现内存溢出的情况。对于简单的基础数据类型（如int）释放数组时不会出现内存溢出，但是对于类对象数组来说，如果类中包含有在堆中分配的数据成员，则必须使用delete []来释放new运算符分配的数组空间。

C++中的高级数据类型

结构体

结构体一般定义在.h文件中，结构体使用struct进行定义，结构体的大小是所有成员大小之和。

共用体

  • 共用体使用union进行定义，共用体的大小是占据内存空间最大的成员所占的空间见。
  • 共用体可以用来存放几种不同数据类型的成员，但在同一时间只能存放其中的一种，不能同时存放集中，即同一时间只有一个成员起作用。
  • 共用体中起作用的成员是最后一次存放的成员，在存入一个新成员后，原有的成员就会被覆盖，从而失去作用。

函数的使用

  • 函数的声明和定义可以分开，函数的定义可以写在函数调用之前，此时函数的定义视为函数的声明。
  • 函数的声明可以不写参数的名称但是一定要指定参数个数和参数类型以及返回值类型。
  • 如果函数的参数是指针，而且只做输入使用，为了防止该指针在函数体内被修改，应该在类型前面加const。
  • 使用默认参数时，没有默认值的参数前面的参数必须也不能有默认值，即有默认值的参数应该放在参数列表的最后面。
  • 形参可变的参数声明：int show(int i, …);如何在函数体中取得所有的可变参数？
  • 使用inline关键字定义内联函数。编译成功的函数。
  • 形参是变量的地址，使用指针可以更改实参，形参相当于是实参的别名。
  • 将数组作为函数的参数时，传递的是数组的起始地址。
  • 函数的重载可以使一个函数名称多次使用，编译器会根据参数的类型和个数区分重载函数，但是不能通过函数的返回类型区分。
  • 声明函数指针时，指针的返回类型要与函数相同，参数类型也要相同，这样才能将指针变量指向函数。
  • 在类内定义的成员函数，被隐式的制定为了内联函数，但是如果成员函数的定义不是在类的内部，而是在类的外部进行定义的，则编译器并不是默认将其按内联函数处理的，这是需要使用inline关键字进行声明，此时，必须将类的声明和类的成员函数定义放在同一个文件里。

面向对象部分

类

  • 一个类里面包含成员变量和成员函数。
  • private表示只能在类内部使用，public表示可以在类外使用，protected表示不能在类和派生成员外使用。
  • 在声明类的时候，成员变量不能进行初始化。
  • 一个类中只能有一个默认的构造函数，也就是说，可以不用参数而调用的构造函数只能有一个。并且如果声明了一个全部参数都指定了默认值的构造函数，就不能再声明重载构造函数了，因为那样会产生歧义。
  • 有时候会用到很多的对象，他们的变量和函数都是一样的，编译器提供了默认复制构造函数。使用默认复制构造函数的时候，使用传址的方式比传值的方式更好，这种方式不用复制整个对象。
  • 析构函数是在对象被撤销后程序结束之前清除并释放所分配的内存。析构函数与类同名，只是前面有一个“~”号。析构函数没有参数也没有返回值。由于析构函数没有参数，所以析构函数不能被重载，一个类只能有一个析构函数。一般情况下，调用析构函数的顺序与调用构造函数的顺序相反。

继承

  • 继承时如果继承基类时没有指定继承的属性，则默认情况下是私有继承。
  • 在派生类中定义一个基类同名的成员函数，派生类生成一个对象，调用对象的成员函数，该成员函数是派生类的成员函数，这叫做成员函数的覆盖。
  • 虚函数可以使用相同的代码实现不同功能调用。虚函数的声明和普通函数成员函数相似，不同之处就是在前面加上一个关键字virtual。
  • 纯虚函数的一般形式是virtual 函数类型 函数名称(参数列表) = 0。纯虚函数没有函数体，在基类中不需要对纯虚函数进行定义。后面的=0只起到形式上的作用，用来说明这个虚函数是纯虚函数。
  • 如果一个类中声明纯虚函数，而在派生类中却没有重新对该函数进行定义，则纯虚函数在派生类中仍然为纯虚函数。
  • 静态成员变量，即使没有生成对象，也会为类的静态成员变量分配空间，静态成员变量是在程序编译的时候分配空间，在程序结束的时候释放空间。
  • 初始化静态成员变量要在类的外面进行。
  • 为了让程序符合封装的理念，不应该直接操作静态成员变量，应该吧静态成员变量设置为private，然后提供静态成员方法操作静态成员变量。
  • 普通成员函数和静态成员函数的区别：普通成员函数在参数传递时编译器会隐式的传递一个this指针，通过this指针来确定调用类产生的是哪个对象。但是静态成员方法没有this指针。由于静态成员方法没有this指针，所以不知道应该访问哪个对象中的数据，因此程序中只能用静态成员方法访问静态成员变量，不能使用静态成员函数访问类中的普通变量。
  • 友元函数可以是普通函数，类的成员函数，使用friend关键字进行声明，当类中声明了友元函数，则该函数能够访问类的所有成员不受private限制。
  • 类也可以使用friend修饰，变成一个友元类。
  • 友元关系是单向的，不能传递，不能继承。比如如果B类是A类的友元类，不等于A类是B类的友元类；如果A是B的友元，B是C的友元，则A和C不存在友元关系；如果A是B的友元，C派生与A，不意味着C是B的友元。
  • 常成员变量使用const关键字进行声明，常成员变量的值是不能改变的。如：const int height;
  • 常成员函数使用const关键字进行声明，常成员函数只能对类中的成员变量进行引用，不能对其进行修改。如：void showMsg() const;
  • 常成员函数不能调用另一个非const成员函数。
  • 运算符重载的关键字是operator，运算符最大的用处就是本来这些运算符只能用于标准类型的运算，但在面向对象的程序中，重要的基础是类和对象，并且允许用户自己定义新的类型。
  • c++关于运算符，有如下限制：
    1. 不能定义新的运算符，只能重载现有的。
    2. 不能改变原运算符的操作数的个数。
    3. 不能改变原有运算符的优先级。
    4. 不能改变原有运算符的结核性。
    5. 不能改变原有运算符的语法结构。
    6. 不能重载”::”,”?”,”:”和”.”。

2.c++中”.”、”->”、”::”的区别是什么？

A.B中A为对象或者结构体。

A->B则A为指针，->是成员提取，A->B是提取A中的成员B，A只能是指向类、结构、联合的指针。

基本功考察题目

1.交换两个数，不使用多余的空间。

思路：使用异或操作运算符即可。

2.左移、右移操作，表示什么？

答案：左移表示乘以2，右移表示除以2.

3.c++中，指针的大小是多少？

答案：指针的大小和操作系统有关系，因为指针存储的是对象的地址，在32位操作系统上，地址采用的是32位寻址方式一个地址占32位。如果操作系统是32位的，那么指针的大小就是4字节；如果操作系统是64位的，那么指针的大小就是8字节。

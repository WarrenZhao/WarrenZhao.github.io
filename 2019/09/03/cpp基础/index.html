变量的使用及其作用域

  1. 寄存器变量：寄存器变量用于建议编译器将变量防止在CPU中的寄存器中，以加快访问速度。在定义变量时，使用register关键字表示该变量是一个寄存器变量。寄存器变量不一定保证变量放在寄存器中，也不能保证访问速度，它只是对编译器的一个建议。寄存器变量不能成为全局变量或者静态变量。
  2. 静态变量：在函数体外定义的变量，声明周期随着程序的结束而结束，只执行一次初始化，使用static关键字来定义。
  3. 外部变量：使用extern关键字定义，用于高速编译器在其他文件的全局区域存在一个全局变量，目的是在当前的文件中能够访问这个全局变量。
  4. 自动变量：使用auto关键字进行声明。局部变量默认状态下就是自动变量。自动变量只能在函数内使用，定义全局自动变量是非法的。
  5. wchar_t型变量：字符类型，此类型的长度为16字节，char的长度是8个字节，char字符类型无法完全包括很多的外文字符集。

引入头部文件的方式和区别

第一种：“”

该种方法引用的头文件一般是自定义的头文件，引用的是程序目录相对路径中的头文件。

比如：项目的目录是在D:\Program\tmp\，则#include “my.h”引用的就是D:\Progam\tem\my.h这个文件

第二种：<>

该种方法引用的是编译器的类库路径里的头文件，无论项目的目录是哪个，只要编译器路径不变更即可。

栈和堆

对于c++来说，程序中的数据主要有两种存储形式，一种是栈存储方式，一种是堆存储方式。栈存储通常用于存储占用空间小、生命周期短的数据，例如局部变量和函数参数等；堆存储通常用于存储占用空间大、生命周期长的数据，例如静态变量和全局变量等。

如何在堆中申请空间？

使用new关键字可以在堆中开辟一块空间，但是需要使用delete关键字释放申请的空间否则会出现内存溢出的情况。对于简单的基础数据类型（如int）释放数组时不会出现内存溢出，但是对于类对象数组来说，如果类中包含有在堆中分配的数据成员，则必须使用delete []来释放new运算符分配的数组空间。

C++中的高级数据类型

结构体

结构体一般定义在.h文件中，结构体使用struct进行定义，结构体的大小是所有成员大小之和。

共用体

  • 共用体使用union进行定义，共用体的大小是占据内存空间最大的成员所占的空间见。
  • 共用体可以用来存放几种不同数据类型的成员，但在同一时间只能存放其中的一种，不能同时存放集中，即同一时间只有一个成员起作用。
  • 共用体中起作用的成员是最后一次存放的成员，在存入一个新成员后，原有的成员就会被覆盖，从而失去作用。

基本功考察题目

1.交换两个数，不使用多余的空间。

思路：使用异或操作运算符即可。

2.左移、右移操作，表示什么？

答案：左移表示乘以2，右移表示除以2.

3.c++中，指针的大小是多少？

答案：指针的大小和操作系统有关系，因为指针存储的是对象的地址，在32位操作系统上，地址采用的是32位寻址方式一个地址占32位。如果操作系统是32位的，那么指针的大小就是4字节；如果操作系统是64位的，那么指针的大小就是8字节。

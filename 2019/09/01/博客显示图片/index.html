起因

博客好久不更新了，有一天更新了一篇带有数学公式的博客，为了让公式正常显示，结果……高挂了我的博客，正职找工作期间，感觉面试官访问不了我的博客好像我在说谎一样。于是，趁着今天有时间，我赶紧把博客恢复了，这一恢复不要紧啊，迁出了更多的问题，尤其是图片不能显示的问题……




简直就是拔出萝卜带出泥，一个问题还没有解决，又增加了新的问题……





本着一个码农应该有的钻研精神，我开始了漫长的搜索和尝试……


首先就是博客高挂了，github没办法显示，整了半天，又是删文件，又是改版本，就是不显示……




于是，我祭出大招——重装！


我的博客终于重见天日！

正题

接下来我就开始来解决图片不显示的问题。首先我看了看不显示图片的url，感觉应该是配置文件有问题，修改了url和root后无果，放弃此方法。在这里要吐槽一下官方文档，这个说明真的是太简单了！


然后再次翻阅官方文档和翻阅无数网页，最终我的解决方案是：

  1. 修改根目录下_config.yml文件中的post_asset_folder: false为post_asset_folder: true；
  2. 修改根目录下package.json文件中的hexo-asset-image": "^1.0.0为hexo-asset-image": "^0.0.1；
  3. 在根目录下执行npm install hexo-asset-image --save；
  4. 编辑带有图片的博客hexo clean && hexo g -d && hexo s尝试查看效果；

总结

图片不能显示的主要原因是markdown语法引用图片需要hexo-asset-image插件的支持，但是1.0.0版本的插件很有问题！

然而，当我解决完了本地问题，发现github上的显示效果和本地一毛一样！





经过这一下午的折腾，我成功熟悉了hexo的各种语法clean、g、d、s无限循环当中，同时成功戒掉抖音一下午。





在查阅资料期间，发现别人写博客也有IDE啊！好工具是第一生产力，不过我觉得我的MacDown就挺好的，哈哈。祝大家早日解决自己的问题！

线程与进程的区别

每一个进程都是有程序入口、顺序执行序列和程序的出口的。但是线程不能够独立执行，因此线程是受到应用程序控制的。

每一个线程都有他自己的一组寄存器，成为线程的上下文，该上下文反映了上次运行该线程的CPU寄存器的状态。

指令指针和堆指针寄存器是线程上下文中两个最重要的寄存器，线程总是在进程得到上下文中运行的，这些地址都用于标志拥有线程的进程地址空间中的内存。

进程和线程的区别如下表：

对比点 进程 线程
系统层面 操作系统资源分配的基本单位，拥有自己的独立的地址空间，每启动一个进程，系统就会为他分配地址空间，简历数据表来维护代码段、堆栈段和数据段 任务调度和执行的基本单位，共享进程中的数据
通讯层面 进程之间通信需要以IPC的方式进行 同一个进程下的线程共享进程的所有变量
资源开销 拥有独立的程序上下文（代码和数据空间），程序之间切换开销大 可以看做是轻量级的进程，统一类线程共享代码和数据空间，每个线程有自己独立的运行栈和程序计数器，线程之间的切换开销小
程序执行 操作系统中能同时运行多个进程（程序） 同一个进程（程序）中有多个线程同时执行（通过CPU调度，在每个时间片内只有一个线程执行）
内存 系统会为每个进程分配不同的内存空间 除CPU外，系统不会为线程分配内存（线程所使用的资源来自其所属进程的资源），线程组之间只能共享资源
关系 可以拥有多个线程 只能属于一个进程
安全性 进程与进程之间不会互相影响，即一个进程挂了，不会影响其他进程，因为每一个进程都有独立的地址空间 多线程程序中，只要有一个线程死掉，整个进程也就死掉了
使用场景 对资源的管理和保护要求高，不限制开销和效率时，使用多进程 要求效率高，频繁切换时，资源的保护和管理要求不是很高时

Python进程

multiprocessing

Unix/Linux操作系统中，提供了fork()函数进行系统调用，使用os模块下直接调用fork()函数即可。但是Windows是没有fork()函数的，可以使用Python的multiprocessing模块进行多进程编程，该模块提供Process类。使用进程池的方式可以批量创建子进程。进程池的默认大小是CPU的核数。进程间通信是通过multiprocessing模块中的Queue和Pipes等是现实的。

  • returncode此进程的退出码。

异常

subprocess模块中的异常类的基类是SubprocessError,其子类包括TimeoutExpired等待子进程的过程中发生超市被抛出、CalledProcessError当一个被check_call()或check_output()函数运行的子进程返回了非零退出码时被抛出。在使用Popen类时，如果尝试执行一个不存在的文件时会被抛出的异常是OSError，如果调用时有无效的参数时，则一个ValueError将会被抛出，check_call()或check_output()函数运行的子进程返回了非零退出码时将抛出CalledProcessError，所有接受timeout形参的函数与方法，将会在进程退出前超时到期时抛出TimeoutExpired。

在多核CPU下由于有GIL（Global Interpreter Lock）的存在，导致每个CPU同一个时间只能执行一个线程，对于CPU密集型代码（循环、计算等）Python下的多线线程对CPU密集型代码并不友好。因此CPU密集型代码推荐使用多进程进行处理，因为每个进程拥有自己独立的GIL。

对于IO密集型的程序，还是推荐使用多线程的。

Python线程

Python线程总述

多线程类似于同时执行多个不同的程序，多线程有很多的有点：

  • 使用线程可以把占据长时间的程序中的任务放到后台处理；
  • 有可能加快程序的运行；
  • 可以用在具有等待任务的地方，比如用户输入、文件读写和网络收发数据，在上述这些任务中，可以减少对内存资源的占用。
    Python中线程两种方式：
  1. _thread模块
  2. threading模块

其中，_thread模块是低级模块，threading模块是高级模块，threading模块对_thread模块进行了封装。大多数情况下，我们只需使用threading高级模块即可。

线程同步

创建线程锁的方法是：
lock = threading.lock()

获取锁的方法是：
lock.acquire()

释放锁的方法是：
lock.release()

在编写线程同步代码的时候，一定要使用try...finally来确保锁一定会被释放，否则造成其他线程一直等待的问题。另外，在存在多个锁的情况下，不同的线程持有不同的锁，并试图获取对方持有的锁时，可能会造成死锁，导致多个线程全部挂起。

总述

一个语法，三个特性，四个用法，一个争论。

lambda表达式的用法即匿名函数。

这里的lambda argument_list:expression表示的是一个函数。这个函数叫做lambda函数。

三个特性

  • lambda函数是匿名的：通俗的讲就是没有名字的函数。
  • lambda函数没有输入和输出：输入是传入到参数列表argument_list的值，输出是根据表达式expression计算得到的值。
  • lambda函数一般功能简单：单行expression决定了lambda函数不可能完成复杂的逻辑，只能完成非常简单的功能。

四个用法

由于lambda语法固定，所以本质上只有一种用法，即定义一个lambda函数。在实际使用中，可以根据场景不同，将lambda函数的用法扩展为一下几种：

  • 将lambda表达式赋值给一个变量：add = lambda x,y:x+y
  • 将lambda表达式赋值给其他函数：time.sleep=lambda x:None在此代码之后如果调用time.sleep(3)，程序不会休眠3秒，而是什么也不做。
  • 将lambda函数作为参数传递给其他函数：sorted([1,2,3,4,5],key = lambda x:abs(5-x))该函数含义为按照列表中数据与5的距离进行排序。
  • 将lambda函数作为其他函数的返回值返回给调用者，例如return lambda x,y:x+y返回一个加法函数。在某个函数内的函数叫做内部函数或者嵌套函数。对应的，将包含内部函数的函数成为外部函数内部函数能够访问外部函数的局部变量，这个特性是闭包编程。

一个争议

在Python社区中，支持方认为使用lambda编写的代码更加紧凑，更加“pythonic”；而反对方认为lambda函数能够支持的功能十分有限，不支持多分支程序（如：if…elif…else…和异常处理try…except…finally…），并且函数的功能被隐藏，需要花费更大的理解成本。

Python中的下划线

Python中的下划线一般会有以下五种情况：

  • 单前导下划线：_var
  • 单末尾下划线：var_
  • 双前导下划线：__var
  • 双前导和末尾下划线：var
  • 单下划线：_

一一介绍五种下滑线的用法。

单导线下划线

根据PEP 8中的说明，下划线前缀的含义是告知其他程序员：以单个下划线开头的变量或方法仅供内部使用。另，在使用通配符从模块中导入所有名称时，以单下划线开头的方法不会被Python导入，如果想要Python将带有前导下划线的名称也能够导入，需要模块去自己定义空值导入行为的all列表。建议坚持常规导入。

单末尾下划线

根据PEP 8中的说明，单个末尾下划线是用来避免与Python关键字产生命名冲突的。如果一个变量的最适合的名称已经被一个关键字所占用，可以使用但末尾下划线命名，通过单末尾下划线解决命名方面的冲突。

双前导下划线

Python中没有类似于c++和java中的public和private关键字，为了实现类似于私有变量的特征，只需要在变量名或者函数名前面加上“__”两个下划线，那么函数或者变量就会成为私有的了。但其实不难发现其实Python只是懂了一下手脚，把双下划线开头的变量进行了改名而已。实际上可以在外部使用“_类名__变量名”的方式即可访问以双下划线开头的“私有变量”了。

注意：Python目前的私有机制其实是伪私有，Python的类是没有权限控制的，所有变量都是可以被外部调用的。

如果想进一步了解Python的双前导下划线可以参看这篇博文Python中的下划线

双前导和双末尾下划线

由双前下划线前缀和后缀包围的变量不会被Python解释器修改。另外由双下划线前缀和后缀包围的方法叫做魔法方法，最好避免在自己的程序中使用双下划线开头和结尾的名称，以避免冲突。

单下划线

独立的下划线，有时候用作一个变量名称，表示某个变量是无关紧要临时的变量。单下划线用于拆分元祖或者列表，此时的“_”可以当做占位符使用。

Python中的继承

首先明确一点，Python是支持多继承的（虽然大部分面向对象的语言是不支持多继承只支持单继承的），但是，却不推荐在使用Python时多继承，单继承在使用的时候思路清晰，同时能够避免不必要的麻烦。因为：如果一个类存在多继承关系，并且多个父类中有同名的方法，那么在排在前面的父类的方法会“遮蔽”后面的方法。出现示例代码中的原因是因为什么呢，这里就不得不提到Python中的MRO方法搜索顺序。比如：

继承可以让子类从父类中获取属性和方法，从父类继承，可以直接拥有父类的属性和方法，这样可以减少代码量，多复用。关于继承的特殊属性和方法有：

  • __base__:类的基类
  • __based__:类的基类元组
  • __mro__:显示方法查找顺序，基类的元组
  • mro()方法：显示方法查找顺序，基类的元组
  • __subclasses__:类的子类列表

继承中父类的方法

子类可以继承父类的方法，也可以重写父类的方法。子类也可以在继承父类方法的同时，对方法进行重构。这样一来，子类的方法既包含父类方法的特性，同时也包含子类自己的特性。

对于继承中的__init__方法，如果在子类中没有重写该方法，实例化子类时，会自动调用父类定义的__init__方法；如果子类重写了__init__方法，实例化子类的时候，就不会调用父类已经定义的__init__方法，此时如果需要调用父类的__init__方法，可以使用super关键字，显示的调用父类的__init__方法。

Python中的拷贝

对于Python而言，Python的一切变量都是对象，变量的存储，采用了引用语义的方式，存储的知识一个变量的值所在的内存地址，而不是这个变量的值本身。我们在看两个对象是不是同一个对象时，可以使用id()这个函数查看其内存地址。

浅拷贝

浅拷贝会创建新的对象，其内容是原对象的引用。浅拷贝有三种形式：

  • 切片操作:list_b = list_a[:]
  • 工厂函数:list_b = list(list_a)
  • copy模块中的copy函数:list_b = copy.copy(list_a)

使用id查看list_a和list_b可以看到他们的地址已经不同了，但是查看list_a和list_b中的元素的地址id(x) for x in list_a和id(x) for x in list_b可以看到，二者包含的对象的地址相同，也就是说：浅拷贝只拷贝了一层。

深拷贝

深拷贝只有一种形式：copy模块中的deepcopy函数，其拷贝了对象的所有元素，包括多层嵌套的元素。

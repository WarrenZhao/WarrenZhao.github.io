我在学习的时候向来是“求甚解”，至少要知道这个事情如果要是这样做，原因是什么，是怎么发展来的，理解了其存在的原因和发展的逻辑，那么我就能理解该事物当下的状态，并且从内心接受它存在的这么一个事实。那既然这样，就先总结一下FasterRCNN的进化史吧。

RCNN

主要算法步骤：

  1. 原始图片通过Selective Search提取候选框，约有2k个；
  2. 候选框缩放成固定大小，并输入到cnn内进行特征提取；
  3. 对候选框提取出的特征，经过两个全链接，使用分类器进一步判别是否属于一个特定类（采用的是SVM算法）；
  4. 对于属于某一特征的候选框，使用回归器进一步调整其位置。

FastRCNN

主要算法步骤：

  1. 在图像中确定候选框（使用Selective Search大概能够生成2k个）；
  2. 整图输入到CNN，得到feature map；
  3. 找到每个候选框在feature map上的映射patch，将patch作为每个候选框的卷积特征输入到SPP layer（Spatial Pyramid Pooling，即空间金字塔池化）之后的层；
  4. 对候选框中提取的特征，使用费雷器判别是否是属于一个特定类；
  5. 对属于某一个特征的候选框，用回归器进一步调整其位置。

流程图如下：

SPP特点是解决了一般的cnn网络需要输入固定尺寸的限制，因此不需要对图像进行crop或者是warp操作，也就避免了数据丢失或者几何失真的潜在问题，实现了多尺度输入但是输出尺度一致的效果。

FasterRCNN

思路

主要算法思路：

  1. 整图输入到CNN，得到feature map，
  2. feature map输入到RPN，得到候选框的特征信息；
  3. 对候选框中提取的特征，使用费雷器判断是否是属于某一个特定类；
  4. 对于属于某一特征的候选框，用回归器进一步调整期位置。

架构图如下：

步骤

有了原文中的框架图，可以将Faster RCNN的步骤总结为如下内容：

  1. conv layers（即卷积层）。Faster RCNN首先要通过一组基础的卷积神经网络提取输入图像image的feature maps。
  2. Region Proposal Networks（常说为RPN网络）。RPN网络用语生成region proposals。该层通过softmax判断anchors属于positive或者negative，再利用bounding box regression修正anchor获得精确的proposals。
  3. Roi Pooling（感兴趣区域池化）。该层有三个输入，第一个输入就是1中提取到的feature maps，第二个是2中的proposals（包含分类信息和位置回归信息），第三个是image_info（图像信息，在输入1之前，需要将最原始的图像reshape一下成为1中能够输入的尺寸，输入之前的图像缩放程度就是image_info)。
  4. Classification。利用proposal feature maps计算proposal的类别，同时再次bounding box regression获得检测框最终的精确位置。

下面是fasterrcnn网络的详细图：

对应两个图，不难发现，下面的图使用的VGG16的结构，既然是VGG16，那么卷积核就是3x3步长是1的同卷积，池化层采用的是最大池化，核大小是2x2，步长为2。最后，从MxN大小的输入图像到feature map，图像在尺寸上，一共的缩放大小是16倍，记住这个16，在RPN阶段需要用到，也就是说feature map的尺寸是(M/16)x(N/16)。

RPN

RPN在检测任务中的用法是，输入图像经过主干网络提取到特征图，特征图上的每个像素点，即为anchor锚点。以每个anchor为中心，人为设置不同尺度和长宽比，即可得到基于anchor的多个anchor box，这些anchorbox是用来框定图像中的目标的。其实RPN最终就是在原图尺度上，设置了密密麻麻的候选Anchor。然后用cnn去判断哪些Anchor是里面有目标的positive anchor，哪些是没目标的negative anchor。所以，仅仅是个二分类而已！

anchor

anchor既然是RPN网络的核心，那么anchor的优缺点有哪些呢？，首先anchor的优点是：

  1. 使用anchor机制产生密集的anchor box，使得网络可以直接在此基础上进行目标分类及边界框的坐标回归；
  2. 密集的anchor box可以有效提高网络目标召回能力，对于小目标检测来说提升还是很明显的。

其次，anchor的缺点是：

  1. 需要设定的超参：尺度和长宽比，这需要较强的先验知识；
  2. 冗余框很多，一张图像内的目标毕竟是有限的，基于每个anchor设定大量的anchor box会产生大量的easy-sample，即完全不包含目标的背景框。会造成正负样本不平衡的问题；
  3. 基于anchor box进行目标分类时，IOU与之超参的设定也是是个大问题。

分类和回归

RPN最主要的功能是输出proposals，首先是feature map要经过一个3x3的卷积和relu层，这样做的目的有人说是为了让结果更加鲁棒，感觉比较牵强。然后分为了两条线，上面一条线经过一个1x1，通道是18的卷积，然后reshape进行softmax分类，目的是得到9个anchor box的positive和negative的得分，因为是9个anchor box然后每个anchor都有两个概率分别对应positive和negative，因此是18维，reshape之后提供给proposal层。第二条线也是经过一个1x1的卷积，通道是36，原因是要对每个anchor box进行回归，每个回归需要（x, y, w, h)四个维度中心点坐标和宽高，因此通道是36正好对应9个anchor box。回归的目标是找到一种变换，使得输入图像映射得到的anchor与ground truth无比接近。其中的变化是线性变化，我比较认同的一种说法是，卷积操作（计算）具有评议不变性和旋转不变性，因此使用线性变换是很合理的。

proposals

做完分类和回归，使用NMS（非极大值抑制）算法，选出前topN的proposals，但是这前topN的只是（positive、negative、x、y、w、h）六种信息，结合image_info和RoI pooling进一步转化，得到proposals对应的feature map。

RoI pooling

RoI pooling层存在的目的就是收集proposals，结合原始的feature map计算出proposal feature maps。那么，为什么要这样做呢？对于传统的cnn网络，要求网络的输入尺寸是固定的，同时网络输出也是固定的，如果输入大小不是固定的，那么可以有两种解决办法：crop（剪裁）和warp（变形）。但是也引入了相应的问题，crop去掉了图像的部分信息，warp改变了图像的原始尺寸。为了不产省这样的问题，RoI pooling借鉴了图像金字塔的思想，具体做法就是：

  1. 先将proposals的坐标映射到feature map的表示空间，让proposals表示的bounding box找到feature map对应的坐标。
  2. 将每个proposals对应的feature map区域划分成pooled_w x pooled_h的网络；
  3. 对2中的每一个网格进行max pooling（最大池化）处理，一个网格代表一个输出值，由于所有的feature map都是pooled_w x pooled_h这样的尺寸，因此所有的输出都是pooled_w x pooled_h的尺寸。

Classification

这部分主要是利用proposal feature maps，通过全连接层和softmax计算每个proposal具体的cls_prob概率；同时利用bounding boxregression获得每个proposal的位置偏移，用于更加精确的目标检测框。

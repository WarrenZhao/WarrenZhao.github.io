概述

语法糖并不是Python特有的，语法糖是指计算机语言中添加的某种语法，这种语法对语言的功能没有影响，但是更方便程序员的使用。可以简单的把从面相过程到面相对象理解成一种语法糖。语法糖为我们的编程带来了方便，是一种便捷的更加易读易用的编程方式，编译器会帮我们进行转换；而且可以提高开发效率，性能上也不会带来损失。当然，在我实际的学习过程中，尤其是早期学习Python的时候，由于缺少对相关语法的针对学习，导致工程中许多语法糖代码看不懂，比如列表生成式，如果没有针对性的学习过，上网查资料都不知道关键字是什么。有人将Python的语法糖重要程度比作学习java时的设计模式，我个人认为实际工程中Python语法糖应用范围广泛，而在实际学习过程中很少有单独总结python语法糖的，因此自己在博客中做一些小小的总结吧，持续更新中…

切片操作

列表的切片操作是建立原列表的一个拷贝（或者说副本），因此原列表是一个保留的状态，如果想要得到切片操作后的列表，直接赋值给一个变量即可。list切片操作之后返回的是list，tuple切片之后返回的是tuple。切片操作一般可以接收三个参数，每个参数之间使用:隔开，如果没有第三个参数，第二个冒号也不用写。切片操作可以完成for循环的遍历功能，但是语法更加的简洁。

装饰器说明

  • Python中的装饰器是一种可以装饰其它对象的工具。
  • 该工具本质上市一个可调用的对象，所以装饰器一般可以有函数、类来实现。
  • 装饰器本身需要接受一个被装饰的对象作为参数，该参数通常为函数、方法、类等对象。
  • 装饰器需要返回一个对象，该对象可以是经过处理的原参数对象、一个包装且类似原参数的对象；或者返回一个不想干内容（通常不建议）

装饰器的应用

  • 用于修改对象的装饰器
  • 函数装饰器
  • 类装饰器
  • 类方法装饰器
  • 特殊装饰器（类静态副属性装饰器）

装饰器的应用

一般比较常见的场景如下：

  • 日志记录
  • 权限验证
  • 单例模式
  • 竞争资源管理

生成器

使用列表生成式的方式创建列表，需要先在内存中开辟一个空间，列表的大小受到了内存的限制，因此列表的容量肯定是有限的。如果创建一个超大的列表，但是仅仅需要访问少量元素，即使内存能够满足列表的空间要求，也造成了内存空间的极大浪费。为了解决这个问题，Python提供了一边循环一边计算的机制：generator（生成器）。

创建一个生成器有很多方法，本文上面提到的生成器表达式就是一种，但要注意生成器表达式和列表生成式的区别：列表生成式使用的是中括号，生成器表达式使用的是小括号。

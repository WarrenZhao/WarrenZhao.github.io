Struts2框架简介

Struts2是一个基于MVC设计模式的Web应用框架，它实质上相当于一个servlet，在MVC设计模式中，Struts2作为控制前来建立模型于是图的数据交互。

Struts2框架的优点

  1. POJO表单及POJO操作
  2. 标签支持
  3. AJAX支持
  4. 易于整合
  5. 支持使用模板生成视图
  6. 插件丰富
  7. 为调试和配置应用程序提供综合的性能分析
  8. 易于修改标签
  9. 促进减少配置
  10. 为多中视图选项（JSP、Freemarker、Velocity、XSLT等）提供技支持

Struts2框架的缺点

  1. 更大的学习曲线
  2. 文档缺乏
  3. 不够透明

Struts2 MVC框架

模型（Model）-视图（View）-控制器（Controller），通常简称MVC，是一种开发web应用的软件设计模式。该软件设计模式由一下三部分组成：

  • 模型——属于软件设计模式的底层基础，主要负责数据维护。
  • 视图——这部分是负责向用户呈现全部或者部分数据。
  • 控制器——通过软件代码控制模型和视图之间的交互。

MVC区分了应用程序的逻辑层和用户界面层，并支持开发关注点的分离。在MVC模式下，控制器接收了来自所有应用程序的请求后，调用模型去准备视图所需要的数据，然后视图使用控制器提供的数据最终生成一个可视的响应。

Struts2第一个程序Hello World

  1. 在src目录下面配置struts.xml。
  2. 在web目录下面新建hello.jsp文件。
  3. 配置web/WEB-INF下面的web.xml文件。
  4. 启动程序，路径为localhost:8080/项目名/actionname。

原理

request -> tomcat -> webapplication -> namespace -> action -> result -> return

Struts2解决的问题：将客户的请求和展现（视图）分离。

namespace

  1. package name=”front”表示前台。
  2. namespace=”/front”定义了访问路径，namespace=””只要没有找到精确action的name，就全都返回给这个action。
  3. 表示返回的结果，默认是success。

拷贝项目的时候，需要更改web-contact的路径。

引入项目的时候，要注意拷贝所有的文进。

如何调价JRE的lib。

Action

当用户进行请求的时候，返回的视图可以由用户自己定义，在struts.xml文件中进行定义。

struts2每次访问的时候，都会new一个新的action对象，服务器的内存优化不用考虑。

当不对class进行配置的时候，用的是ActionSupport对象的excute方法。

访问过程

request -> tomcat -> webapplication -> namespace -> action（发现是对象，对象在Struts2访问的时候就new了）->执行对象的excecute()方法 ->根据返回值返回相应的jsp -> result -> return

action中的class执行的机制

  1. 在Action没有配置，执行ActionSupport对象的excecute方法。
  2. 定义类实现action接口，调用其execute方法。
  3. 定义类继承ActionSupport类，这种方法是最常用的，好处在于可以直接使用Struts2封装好的方法。

路径问题的说明

Struts2中的路径是根据action的路径而不是jsp路径来确定的，所以尽量不要使用相对路径。

解决方法：1.使用jsp定义绝对路径；2.使用标签。

ActionMethod_DMI动态方法调用

  1. 在action中配置method属性，在类中将方法的返回值置为String。（不推荐使用）
  2. 在action中不配置method，在url中通过!进行方法指定。

ActionWildcard_通配符

  1. Struts2的通配符是*，一般用在标签中的name属性和method属性。
  2. 一定要遵守“约定优于配置”。
  3. 不论有多少个*，计数要从1开始。
  4. 如果有多个action都符合，优先匹配精确匹配的，非精确匹配的，按照顺序匹配。

注意：使用通配符可以简化配置！

用action的属性接受参数

部署注意事项：jdk编译版本，tomcat选择的jdk版本（如果有设置）

接收参数的方式：

  1. 和Action指向的class中的成员变量一一对应，原理是通过getter、setter进行赋值。
  2. 使用最多</fontDomainModel接收参数Model（域模型），在Action指向的类中，需要定义get、set方法，不需要new对象。如果遇到了参数和域模型有区别的时候，不适用了。这时候可以使用VO或者DTO。举例：用户名、密码、密码确认，用户的类只需要用户名和密码，存到数据库中。VO可以增加密码确认的成员变量，用VO生成用户。
  3. ModelDriven（不常用，重点学习程序设计的思想）。new出来了action之后，Struts2发现实现实现了ModelDriven接口，然后可以调用getModel（）方法，然后可以调用User的setName（）方法。

DTO：Domain transfer object。后台对象和传输参数对不上的时候用。

Struts2中的控制器：action；视图：JSP；M：model。由控制器来降低耦合度。其中Strtus2重点是关注控制器。

中文参数的处理

如果有中文，就尽量不用get方法。

在Struts2的Struts标签中添加一个标签<constant name="struts.i18n.encoding" value="GBK" />就可以解决中文编码的问题，如果不生效可能是框架bug或者试一试filter的办法解决。

在实际的工作中，自学的方法一般是1.搭建hello world然后一点一点网上面加东西，测试，学习。Struts2的一手资料在官网上面吧。

简单的数据验证

  • 使用Struts2的表亲需要在JSP页面中加入<%@taglib url="/struts-tages" prefix="s" %>标签。
  • <s:property>标签，能取到值栈，actioncontext中的内容，取到的是一个map，通过OGNL表达式取得相应的值。
  • <s:debug>标签能够展示很多值的信息，帮助编译。

访问Web元素

request、session和application:

  • 通过ActionContext可以获取request、Session和application。
  • 实现RequestAware、SessionAware和ApplicationAware接口。(实际中使用的）

HttpServletRequest、HttpSession和ServletContext:

  • 通过ServletActionContext得到HttpServletRequest，通过HttpServletRequest得到HttpSession，通过HttpSession得到ServletContext。
  • 实现ServletRequestAware接口。

DI：dependency injection，依赖注入。
IoC：Inversion of Control，控制反转。
一般来说两个概念等同。

类的成员变量依赖框架将成员变量的值注入，类的成员变量将控制的权限交给Struts2框架进行控制（初始化）。

模块包含

Struts2的xml中使用标签将别人的xml的配置包含进入xml文件中。

默认Action

使用标签标签，当Action不存在是访问默认Action。

总结

  • 实现Action最长用的方式：从ActionSupport继承。
  • DMI动态方法调用!。
  • 通配符配置*,{1},{2}。
  • 接收参数的方法（一般用DomainModel接收）。
  • 简单参数验证addFieldError，这时一般不适用Struts2的UI标签。
  • 访问Web元素：1.Map类型;2.原始类型。（IoC方式依赖Struts2方式）。
  • 包含文件配置。
  • 默认action处理。

Result配置

result类型

  • 默认是dispatcher即运用服务器跳转到视图。
  • redirect跳转到视图。
  • chain跳转到action。
  • redirectAction跳转到action。
  • 其他的知道即可，重点是前面两种。

其他：服务器跳转（地址不变）和客户端跳转（地址改变），如果要跳转到别的包，需要用param指定namespace。

Global_Result全局结果集

package标签中的extends属性，表示继承关系，不用写太多重复的action。

struts-default.xml中有很多的的定义。

dynamic_result动态结果集和带参结果集

$符号表示取得值栈中值。不常用。

forward过程，多个action共享同一个值栈，而一次request只有一个值栈。

redirect的过程中，有两次request也就有两个值栈，传参的时候要在原定义的标签之后添加参数，这个时候在值栈中取不到值，需要从actioncontact中取值。

OGNL

OGNL：Object graph navigation language.
标签里面的属性，在双引号部分的是OGNL表达式。

  • user.xxx只有传递了参数才会构造。
  • 想要初始化Domain Model，可以自己new也可以传参，但是model必须要提供参数为空的默认构造方法。
  • 访问值栈中对象的普通方法，直接调用。
  • 访问静态方法格式+配置（需要进行配置）。
  • 访问静态属性。
  • 访问对象的构造方法。
  • 访问集合（list和数组方式一致）。users.{age}将users的age拿出来生成一个新的list。访问map时的写法dogMap.dog101。
  • 投影（过滤）。users.{?#this.age==1}.{age}或者users.{^#this.age>1}.{age}或者users.{$#this.age==1}.{age}。
  • []代表访问action对象。当服务器端跳转的时候会出现两个action。

标签

object类的标签双引号中默认是OGNL表达式，如果想取得字符串需要加单引号，代表不需要struts2框架解析。

%{}表示把大括号中的内容强制认为是OGNL表达式。

  • $用于i18n和struts配置文件
  • #取得ActionContext的值
  • %将原本的文本属性解析为ognl表达式，对于本来就是ognl的属性不起作用。
  • 定义变量用var属性
  • if、elseif条件用属性test写判断条件
  • iterator可以遍历集合，status属性注意一下。
  • ui：所有主题定义为simple，fielderror特殊处理，自己控制其他标签的展现。

####异常处理

方法：1在代码中加入throw(e)，在action中定义result出错返回的界面。

拦截器

类似于tomcat中的Filter，在invoke方法前后会有一些操作。

定义新的interceptor一般是加在原来的interceptor的基础上。

应用：1.替换敏感字符；2.权限检查等。体现了AOP（面向切面的编程思想）。

权限框架：acegi或者spring security，不要重复发明轮子。

程序员在技术达到一定地步的时候，应该专注分析和业务。

使用token拦截器控制重复提交的问题

很少用，因为有很多其他的办法解决。

防止重复提交的办法：

  1. 使用post方式提交数据。
  2. struts2中使用token。（原理：生成随机数放入session）提交时，验证随机数，交一次之后废弃。
  3. 只需要在struts.xml中添加和标签即可。
  4. 并不能解决重定向之后再提交的问题。

类型转换

在http传参数的时候，类型都是String类型。

struts2转换，只需要按照其固定格式再使用struts2相应的标签即可。传多个值（比如checkbox）使用set、list或者数组后台接收即可。map传参使用较少。对于特殊对象（如没有set方法），可以自定义转换器进行转换。

自定义转换器继承DefaultTypeConverter即可。

注册转换器的方式：

  1. 注册给action，和用到转换器的action放到同一个包，命名是Action名-conversion.properties，将需要转换的名字写到其中。
  2. 全局转换器xwork-conversion.properties(名字固定）
  3. 注解

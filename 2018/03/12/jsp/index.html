程序的封装

如果发现一段代码一直写，可以考虑去封装这段代码。
常用的配置，可以进行封装。另外，可以以重载的方式进行封装。

Note: 代码不需要每次重新写一遍，可以使用现有的方法。
最后无论如何都要关掉的，如果有finally语句，最好写在finally语句里面。

首先先完成基础方法。

javabean

  • 广义的javabean=普通的java类
  • 狭义的javabean=符合Sun JavaBean标准的类。
  • 在Servlet中使用Bean和在通常程序中使用Bean类似。
    • 属性第一个字母必须小写。
    • 一般具有getters和setters
    • Bean不具有GUI表现
    • 一般是来实现某一业务逻辑或取得待定结果

Servlet特性

  • Servlet2.0
  • Servlet3.0
  • Servlet4.0

JSP

什么是jsp？

  • Java Server Pages
  • 拥有servlet特性与优点
  • 直接在HTML中内嵌JSP代码
  • 先由JSP Engine转换成Servlet代码，借着将它编译成类文件载入执行
  • 只有当客户端第一次请求JSP时，才需要将其转换、编译
  • 优点（拥有Servlet的一切优点）：
    • 优良的性能（优于CGI，PHP，ASP）
    • 平台无关性（与操作系统无关，Web服务器无关）
    • 可扩展性，tag的扩展机制，简化页面开发

可以在HTML中的页面中嵌套Java代码。

WEB-INF文件夹下面不能放html文件，也不能放jsp文件。

JSP编程-基本语法

JSP一共有四种语法：

  • JSP传统语法（<%开头）
  • JSTL标签化语言
  • JSF(好东西，但是当时推广的不好）
  • 其他taglib（如struts）

Note:起名字很重要，比如ASP.NET和ASP并没有什么关系，但是名字这样起，用ASP的人就去学ASP.NET了。

JSP传统语法

Declaration

基本语法：

<%! %>

说明：在此声明的变量、方法都会被保留成唯一的一份，编译之后是一个成员变量，直到JSP程序停止执行。

Note：也可以使用<% %>这样的方式进行声明，但是这样声明编译之后是局部变量。

JSP要求<%! %>声明变量的时候，每次刷新相当于调用Service方法。

Note:当刷新特别快的时候，<%! %>声明的变量可能会出现重新初始化的情况。

Note:不能使用<% %>声明方法，相当于在方法内部声明方法，不对吧！

Scriptlet

基本语法：

<%程序代码%>

注释的写法：

  • html方式的注释<!– –>
  • JSP方式的注释<%– –%>
  • Java方式的注释<% //注释 %>

表达式

基本语法：

<%= %>

=后面必须是字符串变量或者可以被转换成字符串的表达式。每一句代码的结尾不用写;，并且只有一行

Directive

Directive（编译指令）相当于在编译期间的命令。
格式：

<%@Directive 属性=”属性值”%>

常见的Directive：

  • page
  • include
  • taglib

page

JSP Container的沟通方式。

<%@ %>
比较重要的，import、errorPage和isErrorPage

include

将指定的JSP程序或者HTML文件包含进来。

格式：

<%@include file=”fileURL%>

JSP Engine会在JSP程序的转换时期先把file属性设定的文件包含进来，然后开始执行转换及编译的工作。

Note: 在编译指令里面，不能传参。因此，include一般用于不传参的。编译的包含指令不能传参。

Action

Action(动作指令)在运行期间的命令。常见的有：

  • jsp:useBean
    • jsp:setProperty
    • jsp:getProperty
  • jsp:include
  • jsp:forward
    • jsp:param
  • jsp:plugin

jsp:include/jsp:param

用于动态包含JSP程序或者HTML文件等。除非这个指令被执行到，否则他是不会被Tomcat等JSP Engine编译的。格式为：

<jsp:include page=”URLSpec” flush=”true” />

或者

<jsp:include page=”URLSpec” flush=”true” >

    <jsp:param name=”ParamName” value=”paramValue” />
</jsp:include>

Note:标签的开头和结尾要一一对应。

jsp:forward

用于将一个jsp的内容传递到page所指定的JSP程序或者Servlet中处理（URL）。forward之前的url传递的参数，forward之后的jsp能拿到。 也就是说Forward的页面和forward到的页面用的是同一个request。

格式：

<jsp:forward page=”urlSpec” flush=”true” >

<jsp:forward page=”urlSpec”>

    <jsp:param name=”paramName” value=”paramValue />

</jsp:forward>

response.sendRedirect和jsp:forward的区别：

forward跳转过程发生在服务器端。两个页面拿到的request是同一个，因为请求只发生了一次。

response.sendRedirect会先和客户端通讯，然后客户端从新发送请求到新的url，发生了两次请求，request是不一样的。

jsp:useBean

通过使用jsp:useBean，可以在JSP中使用定义好的Bean。

Bean的基本要素：

  • 必须要有一个不带参数的构造器。因为JSP元素在创建Bean时会调用空构造器。
  • Bean类应该没有任何公共实例变量，也就是说，不允许直接访问实例变量，变量名称首字母必须小写。
  • 通过getter/setter方法来读/写变量的值，并且将对应的变量首字母改成大写。

Note: 不要使用裸体类，需要放到包里面。

用法：

<jsp:useBean id=”cb” class=”bean.CounterBean>

    <jsp:setProperty name=”cb” property=”count” value=”23” />

    <jsp:getProperty name=”cb” property=”count” />

</jsp:useBean>

其中，<jsp:getProperty name=”cb” property=”count” />相当于out.println(cb.getCount());

属性：

  • scope
  • type（父类引用指向子类对象时候能用到）

Note:多态的三个必要条件：1.要有继承；2.要有方法的重写；3.父类引用指向子类对象。

jsp:useBean各项参数的含义：

  • id：对象实例名称
  • scope:Bean作用范围，默认为page，对这个jsp页面有效（可以理解为局部变量）
  • class：Bean类名称
  • type:Bean实例类型，可以是本类，或者父类，或者实现的接口，默认为本类。

Scope各项参数的意义：

  • page：仅涵盖使用JavaBean的页面
  • request：有效范围仅限于使用JavaBean的请求
  • session：有效范围在用户整个连接过程中（整个会话阶段均有效）
  • application：有效范围涵盖整个应用程序。也就是对这个网站均有效。

jsp:getProperty和jsp:setProperty的细节问题

jsp:getProperty标签中的param等同于value=”<%=request.getParameter()%>

jsp中可以自动进行转换，比如String转换成double类型。

jsp标签中的property属性，类中可以没有明确的成员变量但是主要有相应的getter和setter就能用。（这里用到了java的反射机制）。

jsp:setProperty 中property=”*”的用法（写法简化）通过一个form设置bean整个的内容。写法用的比较少，很容易出错。但是Struts中用的很多。

jsp显示乱码的问题

  1. 分析经过的环节，页面->提交->set->get。
  2. 检查方法：打印出来即可。

解决方法：

  • 转码。String的getBytes()方法。传输过程中使用的是（“ISO8859_1”）这个编码。（需要知道编码细节）。
  • 使用request方法，即request.setCharacterEncoding(“GBK”)。

jsp内置对象

jsp的常用的对象

  • pageContext当前页面的运行环境，用的也比较少。
  • config一般的指的是web.xml对于jsp编译成功之后的servlet的配置。
  • page，代表当前页面，用处较少。

以上用的比较少，以下比较多

  • out, 最常用的方法println()
  • request，getparameter()
  • response, sendRedirect()
  • cookie
  • session
  • application

jsp和Servlet的通讯

jsp->servlet,直接使用jsp:forward标签
servlet->jsp,getServletConfig().getServletContext().getServletDispatcher().forward/include

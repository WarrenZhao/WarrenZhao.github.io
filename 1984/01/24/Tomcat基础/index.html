背景知识

常用的Web服务器

  1. IIS
  2. Apache
  3. Tomcat

常用的动态网页技术

根据客户端不同的情况，展现不同的内容（在同一个网页中）。

  1. CGI（Common GateWay Interface)最早的动态网页技术；
  2. API
  3. ASP （Active Server Page), 仅仅在微软平台；以进程方式运行；ASP.Net;
  4. PHP (Personal Home Page);
  5. JSP/Servlet (Java Server Page)，以线程的方式运行。

JSP就是一种Servlet。

那么问题来了，什么是Servlet呢？

  • Servlet就是服务器端的一个程序;
  • 用来完成B/S架构下，客户端请求的响应处理；
  • 平台独立，性能优良，能以线程方式运行；
  • Servlet API为Servlet提供了统一的编程接口；
  • Servlet一般在容器中运行；
  • 常见的Servlet容器
    • Tomcat(是一个webserver也是一个Servlet的容器）

Tomcat常见配置

  • Mac设置JAVA_HOME
  • Tomcat debug模式
  • Mac设置CLASSPATH
  • Mac设置默认端口
  • 80端口、8080端口

注意：Tomcat是使用Java编写的，如果有异常，首先尝试读取一下，看看是怎么回事。

Tomcat的目录结构

bin/:一般的脚本文件
conf/:Tomcat配置文件
lib/:类库
logs/:日志目录
temp/:临时产生的文件
webapp/:各种各样的应用程序
work/:Tomcat工作目录（jsp产生的class文件）

Tomcat 9中没有的目录文件
common/:常见的jar包，自己需要写程序需要用到的jar包
server/:服务器所需要的类库目录
shared/:用户应用程序类库目录，目前都是lib吧

Tomcat配置文件

  • server.xml

Tomcat启动时会使用到。
<Connecor>标签可以设置Tomcat的启动端口，但是需要重启Tomcat才能生效。

注意：在对其进行更改的时候，最好先进行一份备份

  • web.xml

定义所有的Web应用的配置

  • tomcat_user.xml

定义了tomcat用户是否可以访问后台
